## 本程序启动参数

参考: [Java 17](https://docs.oracle.com/en/java/javase/17/docs/specs/man/java.html)

```text
# ------------------------------------------------------------------------------------
# 杂项
# ------------------------------------------------------------------------------------

# 不打印所有虚拟机参数
-XX:-PrintCommandLineFlags

# 不打印所有虚拟机参数的默认值
-XX:-PrintFlagsInitial

# 查看所有虚拟机参数的本虚拟机实例的实际值
-XX:+PrintFlagsFinal

# ------------------------------------------------------------------------------------
# 执行引擎
# ------------------------------------------------------------------------------------

# 服务器模式 使用更激进的JIT优化方案 (64位系统默认开启)
-server

# 解释执行与JIT代码混合方式
-Xmixed

# ------------------------------------------------------------------------------------
# 内存 (堆)
# ------------------------------------------------------------------------------------

# 初始堆内存
# 等价于 -Xmx16g
-XX:InitialHeapSize=16g

# 最大堆内存大小
# 等价于 -Xms16g
-XX:MaxHeapSize=16g

# 堆内存中新生代和老年代的尺寸关系 默认值为2, 即1:2
-XX:NewRatio=2

# 伊甸园区与S0/S1的比例关系 默认值为8，即 8:1:1
-XX:SurvivorRatio=8

# 自动调整伊甸园区 S0/S1 区自动分配策略，关闭此策略
-XX:-UseAdaptiveSizePolicy

# 新生代对象经理过n此GC则被保存到老年代
-XX:MaxTenuringThreshold=15

# 是否在伊甸园区开辟线程专属内存空间 默认为开启
-XX:+UseTLAB

# ------------------------------------------------------------------------------------
# 内存 (栈)
# ------------------------------------------------------------------------------------

# JVM栈每个线程的栈占用尺寸
# 等价于 -Xss512k
-XX:ThreadStackSize=512k

# ------------------------------------------------------------------------------------
# 内存 (元数据)
# ------------------------------------------------------------------------------------

# 元数据区初始化大小
-XX:MetaspaceSize=256m

# 元数据区最大内存大小
-XX:MaxMetaspaceSize=1g

# 使用对象压缩指针
-XX:+UseCompressedOops

# ------------------------------------------------------------------------------------
# 内存 (直接内存)
# ------------------------------------------------------------------------------------

# 直接内存最大尺寸
# 默认为0，意思是JVM自行判断
-XX:MaxDirectMemorySize=16g

# ------------------------------------------------------------------------------------
# 内存 (垃圾收集)
# ------------------------------------------------------------------------------------

# 使用G1收集器
-XX:+UseG1GC

# G1收集器期望达到的最大GC停顿指标 默认为200ms
-XX:MaxGCPauseMillis=300

# STW时GC线程数
-XX:ParallelGCThread=8

# ------------------------------------------------------------------------------------
# 内存 (垃圾收集log)
# ------------------------------------------------------------------------------------

# 输出GC日志 (已过时)
-XX:+PrintGCDetails

# 输出GC日志
-Xlog:gc*=info:file=./logs/gc.log

# ------------------------------------------------------------------------------------
# 内存 (heap dump)
# ------------------------------------------------------------------------------------

# 在堆空间内存溢出时生成dump文件
-XX:+HeapDumpOnOutOfMemoryError

# 在FullGC之前生成dump文件
-XX:+HeapDumpBeforeFullGC

# 日志位置
-XX:HeapDumpPath=./logs/Java-HeapDump.hprof
```
